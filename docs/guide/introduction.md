# 产品介绍

## 项目愿景

> 打造一款开源的强大的 Java 调试工具，方便 Java 工程师针对异网环境（容器环境）进行类本地化开发。

在生产环境、容器环境、K8s 集群等"异网"场景下，调试 Java 应用往往面临重重障碍：网络隔离无法直连、代码修改后需要重新构建发布、日志获取困难…… Debug4j 正是为解决这些痛点而生。

## 核心能力

| 能力 | 一句话说明 |
|------|-----------|
| 🚇 **代理穿透** | 容器内任意 TCP 端口映射到本地，SSH/数据库/JDWP 一键可达 |
| 🔥 **源码热更新** | 在线修改源码 → 即时编译 → 秒级生效，无需重启 |
| 🛠️ **进程管理** | JVM 参数/Spring 配置动态调整，进程重载，对象增强 |
| 📜 **日志管理** | 日志级别动态调整，stdout 过滤，实时 tail，zgrep 检索 |
| 📦 **组件增强** | SFTP 文件管理，一键安装 OpenSSH/Arthas，HTTP 代理 |

## 产品流程

### 认证与连接

1. **登录认证**：系统需要登录才能访问，使用 HTTP Basic 认证方式
2. **自动注册**：引入 Debug4j 依赖后会自动创建 **Client 端**（调试引擎）+ **Proxy Client 端**（流量代理）
3. **令牌校验**：Client 注册时携带令牌进行身份验证，并声明自身版本号
4. **版本兼容**：Server 端根据 Client 端不同版本进行兼容性处理

### 节点选择

采用 **三层结构** 组织节点：

```
ServerName（服务名）
  └── NodeIp（节点 IP）
       └── NodeId（节点标识，同一用户进程的唯一标识）
```

### 功能分配

| 功能 | 使用的 Client |
|------|--------------|
| 🚇 代理穿透 | **Proxy Client**（独立进程确保代理不中断） |
| 🔥 源码管理 / 🛠️ 进程管理 / 📜 日志管理 / 📦 组件增强 | **Client** |
| 👑 接管所有调试功能（Child 存在时） | **Child Client** |

## 核心优势

- **零配置接入**：Spring Boot Starter 一键引入，自动注册
- **双模式架构**：线程模式低开销 + 进程模式高隔离，JDWP 场景不阻塞
- **生产安全**：令牌认证 + HTTP Basic 授权，代理端口范围可控
- **容器友好**：支持 Docker/K8s 环境，无需修改网络配置
- **开源免费**：Apache 2.0 协议
